USAGE                                                              *ntf-USAGE*

>
  Usage: ntf [options] [spec-file-or-dir...]

  Options:
    --timeout=MS        kill a worker after MS milliseconds (default: 60000; 0 disables)
    --filter=PATTERN    run only tests whose full name matches the Lua pattern
    --jobs=N            max parallel nvim workers (default: cpu count)
    --shuffle[=SEED]    randomize test order; SEED fixes it (default: time based)
    --test-hook=PATH    run a Lua module providing setup/teardown around each test, in its worker
    --global-hook=PATH  run a Lua module providing setup/teardown once around the whole run, in the launcher process
    --coverage[=FILE]   measure line coverage; write luacov.stats.out (or FILE) and print a summary
    -h, --help          show this help

  With no paths, runs the *_spec.lua files under ./spec.
<

==============================================================================
WRITING SPECS                                              *ntf-WRITING-SPECS*

>lua
  -- The test API is pulled from `require("ntf")` explicitly (no global injection).
  local ntf = require("ntf")
  local describe, it, pending = ntf.describe, ntf.it, ntf.pending
  local before_each, after_each, finally = ntf.before_each, ntf.after_each, ntf.finally
  local assert = ntf.assert

  describe("group", function()
    local value
    before_each(function()
      value = 1
    end)
    after_each(function()
      value = nil
    end)

    it("does something", function()
      finally(function()
        -- runs when this test finishes, whether it passed or failed
      end)
      assert.equal(1, value)
    end)

    pending("not implemented yet")
  end)
<

HOOKS                                                              *ntf-HOOKS*

`--test-hook=PATH` loads the given Lua module in every worker (via `dofile`).
Each test runs in its own worker process, so the module's optional `setup` and
`teardown` functions run once per test — but outside everything the spec itself
defines: `setup` before the spec is built, `teardown` after the worker's test
has run. They are deliberately not named `before_each`/`after_each` — those are
spec hooks around the test body; `setup`/`teardown` bracket the whole worker
instead.
>lua
  -- test_hook.lua
  return {
    setup = function() end,
    teardown = function() end,
  }
<
>sh
  ntf --test-hook=./test_hook.lua
<
A relative path resolves against the working directory (the plugin under test).
An error raised while loading the module or from `setup` is reported as a load
error. A `teardown` error is reported too — as an error entry alongside the
worker's results, so it fails the run without discarding the results already
produced.

`--global-hook=PATH` takes a module with the same contract but runs it once in
the launcher process instead of in every worker: `setup` before any spec file is
loaded, `teardown` after all workers have finished. Use it for state shared by
the whole run — start a server once, build a fixture once — while `--test-hook`
remains the per-test bracket:
>sh
  ntf --global-hook=./global_hook.lua
<
An error raised while loading the module or from its `setup` aborts the run
before any test starts. A `teardown` error is reported after the results, so it
fails the run without discarding them.

Because `setup` runs before the spec is built, it is the injection point for a
debugger: the code under test loads while the debugger is already attached. ntf
has no debugger dependency of its own:
>lua
  -- debug.lua
  return {
    setup = function()
      require("lldebugger").start()
    end,
  }
<
>sh
  ntf --test-hook=./debug.lua --jobs=1 --filter='the test name'
<
Tests run in parallel worker processes whose stdout ntf captures, so to actually
attach a debugger keep it to a single worker (`--jobs=1`, and narrow to one test
with `--filter`). Wiring the debugger transport itself is up to your module.

COVERAGE                                                        *ntf-COVERAGE*

`--coverage` measures line coverage of the code under test while the specs run.
It measures every file under the working directory except the test tree: any
`*_spec.lua` file and the test directory the specs were found in (its top-level
directory under the working directory — `spec/` by default, but whatever path you
pass) are excluded, so anything sitting alongside the specs there (such as cloned
test dependencies) is left out too. A measured file no test executed still shows
up, at 0% (LuaCATS meta files are skipped: they never run by definition). It
needs no extra install: ntf sets a Lua line
hook in each worker, merges the per-worker counts, prints a short summary, and
writes a `luacov.stats.out` (override the path with `--coverage=FILE`):
>sh
  ntf --coverage
<
The built-in summary is intentionally simple (its line classification is a
heuristic). For an authoritative or HTML report, point LuaCov — which ntf does
not depend on — at the same stats file:
>sh
  luarocks install luacov
  luacov          # reads luacov.stats.out -> luacov.report.out
<
Coverage forces the interpreter (`jit.off()`) in each worker so the line hook is
not skipped by the JIT, which makes a `--coverage` run slower than a plain one.

Lua module: ntf                                                      *ntf-ntf*

describe({name}, {fn})                                        *ntf.describe()*
  Define a test group. Its body runs at build time to discover nested
  `describe`/`it`; the body itself is never reported as a test.

  Parameters: ~
    {name} (string) group name
    {fn} (fun()) body that declares nested `describe`/`it`

it({name}, {fn}, {opts})                                            *ntf.it()*
  Define a test case. The body runs at execution time, in its own fresh Neovim
  process. This is not configurable: state never leaks between tests, because
  no two tests ever share a process.

  Parameters: ~
    {name} (string) test name
    {fn} (fun()) test body
    {opts} (NtfItOption?) |NtfItOption|

pending({name}, {fn})                                          *ntf.pending()*
  Mark a test as pending. As a declaration it records a skipped node; called
  inside a running test body it aborts the test as pending.

  Parameters: ~
    {name} (string) pending reason
    {fn} (fun()?) optional body (ignored; pending is never executed)

before_each({fn})                                          *ntf.before_each()*
  Register a hook run before each test in the current group.

  Parameters: ~
    {fn} (fun()) hook body

after_each({fn})                                            *ntf.after_each()*
  Register a hook run after each test in the current group.

  Parameters: ~
    {fn} (fun()) hook body

finally({fn})                                                  *ntf.finally()*
  Register a callback run when the current test finishes (pass or fail).

  Parameters: ~
    {fn} (fun()) callback body

decorate_coverage({opts})                            *ntf.decorate_coverage()*
  Decorate a buffer's sign column with per-line test coverage read from a
  `luacov.stats.out` file (as written by `ntf --coverage`): covered lines are
  marked with the `NtfCoverageCovered` highlight, coverable-but-missed lines
  with `NtfCoverageMissed`.

  Parameters: ~
    {opts} (NtfDecorateCoverageOption?) |NtfDecorateCoverageOption|

is_decorated_coverage({opts})                    *ntf.is_decorated_coverage()*
  Whether `decorate_coverage` is currently drawing on the buffer. Intended for
  a toggle mapping paired with `decorate_coverage`.

  Parameters: ~
    {opts} (NtfIsDecoratedCoverageOption?) |NtfIsDecoratedCoverageOption|

  Return: ~
    (boolean)

Lua module: ntf.assert                                        *ntf-ntf.assert*

equal({want}, {got})                                      *ntf.assert.equal()*
  Assert two values are equal (`==`).

  Parameters: ~
    {want} (any) expected value
    {got} (any) actual value

same({want}, {got})                                        *ntf.assert.same()*
  Assert two values are deeply equal.

  Parameters: ~
    {want} (any) expected value
    {got} (any) actual value

truthy({got})                                            *ntf.assert.truthy()*
  Assert `got` is truthy (not `nil` and not `false`).

  Parameters: ~
    {got} (any) value under test

falsy({got})                                              *ntf.assert.falsy()*
  Assert `got` is falsy (`nil` or `false`).

  Parameters: ~
    {got} (any) value under test

is_true({got})                                          *ntf.assert.is_true()*
  Assert `got` is exactly `true`.

  Parameters: ~
    {got} (any) value under test

is_false({got})                                        *ntf.assert.is_false()*
  Assert `got` is exactly `false`.

  Parameters: ~
    {got} (any) value under test

is_nil({got})                                            *ntf.assert.is_nil()*
  Assert `got` is `nil`.

  Parameters: ~
    {got} (any) value under test

match({pattern}, {got})                                   *ntf.assert.match()*
  Assert `got` is a string matching the Lua `pattern`.

  Parameters: ~
    {pattern} (string) Lua pattern
    {got} (any) value under test

register({name}, {fn})                                 *ntf.assert.register()*
  Register a custom assertion onto ntf's assert object.
  Available afterwards as `assert.{name}(...)` and `assert.is_not.{name}(...)`.

  Parameters: ~
    {name} (string) assertion name
    {fn} (function) factory `function(self) return function(_, args) ... end end` returning the predicate

register_eq({name}, {get_actual})                   *ntf.assert.register_eq()*
  Register an assertion that compares the actual value with `==`.

  Parameters: ~
    {name} (string) assertion name
    {get_actual} (function) maps the leading args to the actual value (last arg is the expected)

register_same({name}, {get_actual})               *ntf.assert.register_same()*
  Register an assertion that compares the actual value with deep equality.

  Parameters: ~
    {name} (string) assertion name
    {get_actual} (function) maps the leading args to the actual value (last arg is the expected)

STRUCTURE                                                      *ntf-STRUCTURE*

NtfItOption                                                      *NtfItOption*

- {timeout} (integer?) per-process timeout in ms, overriding the
    `--timeout` default; an exceeded worker is killed and reported as an
    error.

NtfDecorateCoverageOption                          *NtfDecorateCoverageOption*

- {enable} (boolean?) when `false`, clear the decoration instead of drawing
    it (default `true`).
- {path} (string?) `luacov.stats.out` file to read (default
    `"./luacov.stats.out"`).
- {buffer} (integer?) target buffer (default `0`, the current buffer).

NtfIsDecoratedCoverageOption                    *NtfIsDecoratedCoverageOption*

- {buffer} (integer?) target buffer (default `0`, the current buffer).

NtfAssert                                                          *NtfAssert*

- {no} (NtfAssert) the same assertions with the expectation flipped
- {[string]} (fun(...)) custom assertions registered via `ntf.assert.register*`
